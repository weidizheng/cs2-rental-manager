# CS2 出租选品报告：数据流程与评分方法

版本：2026-07-21 v2
适用报告：刀具出租选品、死亡之手（四代）手套出租选品
维护原则：评分口径、数据限制或平台费率变化时，先更新本文档，再同步更新生成脚本和 HTML 中的简版说明。

## 1. 目标与输出

报告回答四个问题：

1. 能否以合理价格买到；
2. 买到后是否有足够出租现金流；
3. 求购价、出租报价和成交量是否真实；
4. 需要退出时，二级市场是否有足够承接。

当前输出：

- 桌面 `CS2刀具出租选品报告.html`
- 桌面 `CS2四代手套出租选品报告.html`
- 桌面生成器 `generate_cs2_knife_rental_report.py`
- 本方法论文档 `RENTAL_SELECTION_METHODOLOGY.md`

报告只包含公开市场数据和计算结果，不写入 API Token、私钥或项目账号信息。

## 2. 数据源与职责

### 2.1 CSQAQ

[排行榜接口](https://docs.csqaq.com/api-187131776)提供价格、租赁、成交量、在售/求购数量和价格涨跌数据。主要字段：

- 国内各平台最低在售价、最高求购价；
- 悠悠短租/长租报价和当前在租数量；
- Steam 日成交量；
- BUFF 在售/求购深度；
- 7/30/90 日价格涨跌；
- C5、悠悠、IGXE、ECO 的商品 ID、租金和价格。

CSQAQ 的“当前出租数量/在租数量”视为供给和市场覆盖代理，不视为每天完成出租数量。

### 2.2 CSFloat

使用项目只读接口获取：

- 最低在售价；
- 当前最高匹配求购价、数量和附带条件；
- 近期成交历史；
- 官方 USD/CNY 汇率。

近期成交接口当前通常最多返回约 40 条，因此 `24h/72h` 统计可能是截断样本。成交高度集中在同一小时的批量记录会被降权。

### 2.3 项目缓存与费率

`private-data/app.db` 提供项目已经刷新过的市场报价、四代手套分类和出租平台报价；旧版 `market_cache.json` 只在升级时作为一次性迁移来源。
`private-data/configs.json` 提供 C5、IGXE、ECO 等平台的首租/转租费率以及 CSFloat 官方汇率获取所需配置。

悠悠租金只用于判断其他平台租金是否异常，不作为实际出租收入来源。

### 2.4 终端机制资料

- [Valve《死亡之手》公告](https://www.counter-strike.net/newsentry/518615049848225858?l=tchinese)：22 款新手套，终端可通过每周补给获得；
- [Steam 死亡之手终端市场页](https://steamcommunity.com/market/listings/730/Sealed%20Dead%20Hand%20Terminal)：终端为可交易商品，并展示其内容；
- [Genesis 终端报道](https://www.pcgamer.com/games/fps/counter-strike-2-invents-a-new-way-to-monetise-guns-and-hoo-boy-people-seem-to-hate-it-even-more-than-crates-and-keys-im-not-paying-1000-bucks-for-a-skin/)：玩家先看到具体物品和报价再接受或跳过，动态定价公式没有公开。

Valve 没有公布接受率、磨损、稀有度和报价之间的精确公式。因此报告只把动态调价作为不确定性和供给反馈机制，不反推不存在的“官方价格底”。

## 3. 总体处理流程

1. 从 CSQAQ 排行榜拉取目标类别；
2. 从接口阶段排除 StatTrak；
3. 刀具按 1500–4000 元和初筛得分选出 20 个深查候选；
4. 四代手套按官方 22 个英文 `market_hash_name` 建立完整集合；
5. 获取各市场在售、求购、租金、成交量和价格趋势；
6. 用 CSFloat 官方汇率把美元统一换算成人民币；
7. 计算即时买入和挂求购两条入场路径；
8. 校准租金并扣除平台费率；
9. 刀具计算刀型偏好，手套计算终端供给风险；
10. 计算年化、90 日压力收益和综合得分；
11. 输出 HTML，并对表格、卡片、数据完整性和未替换模板变量做检查。

## 4. 入场价格模型

### 4.1 即时买入

```text
即时买入价 = min(国内各市场最低在售价, CSFloat最低在售价)
```

该价格是当前可以立即成交的保守成本，也是“等待挂单是否值得”的比较基准。

### 4.2 挂求购入场

对每个市场读取当前最高求购价，并用统一的 0.10 元最小加价近似成为新一档最高求购：

```text
平台目标求购价 = 平台当前最高求购价 + 0.10元
计划求购价 = min(所有可用平台目标求购价)
```

若计划求购价不低于即时买入价，则使用即时买入，不制造虚假折价。

```text
求购折价率 = (即时买入价 - 计划求购价) / 即时买入价
```

CSFloat 与国内现货的专项比较：

```text
CSF求购对国内折价 =
    (国内最低在售价 - CSF最高求购价 - 0.10) / 国内最低在售价
```

### 4.3 求购成交可信度

CSFloat 求购需要至少 8 条 72 小时成交样本才进行强弱判断：

- 强支撑：成交 P25 与求购价差不超过 3%，且最近成交也在 3% 内；
- 一般：P25 差距不超过 7%，或至少有成交在 5% 内；
- 弱支撑：求购价明显低于近期真实成交；
- 样本不足：不把求购视为高确定性成本。

国内平台没有同等级别成交历史时，使用 Steam 成交量和求购价距即时在售价的距离作为代理，且可信度上限为 75 分。

```text
有效入场优势 = 求购折价分 × (35% + 65% × 成交可信度)
```

因此，大折价但几乎不可能成交，不会自动获得高分。

## 5. 出租收益模型

### 5.1 悠悠租金校准

- 悠悠租金只作为锚点；
- C5/IGXE/ECO 高于悠悠 35% 的部分截尾到 `悠悠×1.35`；
- 低于悠悠的报价保留，因为它可能更接近实际可执行租金；
- 悠悠本身不参加“最佳实际出租平台”的收入选择。

截尾比直接删除高报价更合理：删除会把有租金但报价偏高的商品错误变成零收益。

### 5.2 首租/转租综合费率

基准假设：85% 的付费出租天来自转租订单。

```text
综合费率 = 15% × 首租费率 + 85% × 转租费率
净日租 = 截尾后日租 × (1 - 综合费率)
```

### 5.3 年化情景

展示 80%、85%、90% 三档有效出租率，综合评分使用 80% 保守档：

```text
净年化 = 净日租 × 365 × 有效出租率 / 计划入场价
```

同时展示即时买入价对应年化，防止挂单低价让年化看起来不切实际。

## 6. 90 日压力收益

该指标是统一压力测试，不是价格预测或收益保证。

```text
90日净租金收益 = 净日租 × 90 × 80% / 计划入场价
入场折价兑现 = min(求购折价率, 15%) × 求购成交可信度
趋势扣减 = 65% × max(0, -30日涨跌) + 35% × max(0, -90日涨跌)
```

刀具：

```text
90日压力收益 = 90日净租金收益 + 入场折价兑现 - 趋势扣减
```

四代手套额外扣除终端不确定性：

```text
终端不确定性扣减 = max(0, 终端风险分 - 50) × 0.08
手套90日压力收益 =
    90日净租金收益 + 入场折价兑现 - 趋势扣减 - 终端不确定性扣减
```

## 7. 刀型偏好：为什么不能等同成交量

成交量是最重要的客观代理之一，但不能直接等于玩家偏好：

- 便宜刀更容易高频换手；
- 不同刀型的皮肤数量和存世量不同；
- 新刀型和旧刀型的上市时间不同；
- 投资者套利成交与玩家自用需求混在一起；
- 高价三幻神即使玩家更喜欢，绝对成交量也可能被价格压低。

因此采用家族级综合偏好：

```text
刀型偏好分 =
    45% × 长期偏好先验
  + 35% × 该刀型单款Steam成交中位数分位
  + 20% × 该刀型订单深度中位数分位
```

长期偏好先验只占总评分的 `15% × 45% = 6.75%`，不会让主观认知覆盖真实市场数据。

主要先验层级：

- 顶级：蝴蝶刀、爪子刀、M9 刺刀；
- 高：锯齿爪刀、骷髅匕首、刺刀；
- 中高：流浪者、短剑、海豹短刀、廓尔喀、猎杀者、折叠刀；
- 中低：熊刀、鲍伊、弯刀、系绳、求生；
- 低：穿肠刀、暗影双匕、折刀。

该层级应随长期市场数据变化更新，而不是永久固定。

## 8. 刀具综合评分

```text
刀具总分 =
    25% 出租净收益
  + 15% 出租覆盖
  + 15% 入场折价与成交可信度
  + 15% 转售流动性
  + 15% 刀型偏好
  + 10% 价格稳定
  +  5% 数据质量
```

说明：

- 出租覆盖是当前挂单/在租和多平台报价覆盖，不等同真实出租需求；
- 转售流动性综合 CSFloat 成交数量、成交间隔、成交集中度、价格波动、Steam 日成交和求购承接；
- 数据质量检查 CSFloat 成交、求购支撑、租金平台数量和 Steam 成交是否可用。

## 9. 四代手套的终端风险

### 9.1 可确认事实

- 2026-03-11 发布；
- 共 22 款新手套；
- 死亡之手终端可以出现在每周补给；
- 玩家看到具体物品和报价后决定购买或跳过。

### 9.2 不可确认部分

Valve 未公开动态报价的精确输入、调价步长、时间窗口、个性化规则或价格上下限。社区常见的“接受增加则涨价、拒绝增加则降价”只能作为机制观察，不能当成可精确复现的公式。

### 9.3 市场影响

动态报价形成一条有反馈的供给曲线：

1. 终端报价低于二级市场时，玩家更愿意接受，新增供给加快；
2. 新供给进入二级市场，对价格形成压力；
3. 报价过高时接受率下降，新增供给放慢；
4. 二级市场可能围绕终端获得成本收敛，但该价格不是公开、固定的硬底。

### 9.4 终端风险分

```text
终端风险 =
    25 每周持续供给
  + 15 动态算法不透明
  +  0–10 新系列短历史
  +  0–10 近7日下跌
  +  0–20 近30日下跌
  +  0–10 近90日下跌
  +  0–10 市场在售深度分位
```

风险分越高，终端稳定分越低：

```text
终端稳定分 = 100 - 终端风险分
```

### 9.5 手套综合评分

```text
手套总分 =
    25% 出租净收益
  + 15% 出租覆盖
  + 15% 入场折价与成交可信度
  + 15% 转售流动性
  + 15% 终端稳定
  + 10% 价格稳定
  +  5% 数据质量
```

手套必须按磨损区间比较。久经沙场的 0.15–0.20、0.20–0.28、0.28–0.38 应视为不同质量层级；最低在售价对应的高磨损不能直接套用到低磨损求购。

## 10. 风险标记与排除

刀具初筛会排除或降权：

- 缺少长租/短租报价；
- 长租日租异常高于短租；
- 榜面年化异常；
- Steam 日成交为零；
- 出租挂单/在租为零；
- 国内买卖价差过大；
- CSFloat 成交历史不可用或成交高度集中；
- 求购价缺少近期真实成交支撑；
- 刀型偏好较弱。

手套风险标记：

- 终端供给/调价风险高；
- 近 30 日仍明显下跌；
- 求购成交可信度低；
- 出租挂单样本少；
- 实际平台租金显著高于悠悠锚点；
- CSFloat 成交历史未深查。

## 12. 当前限制与后续改进

1. **真实出租需求不足**：目前主要是挂单供给和项目自身订单，尚无全市场每日完成出租量；
2. **国内求购成交历史不足**：国内挂单的成交可信度仍使用代理；
3. **CSFloat 历史截断**：最多约 40 条近期样本，不适合做长期成交量；
4. **终端报价历史缺失**：尚未建立按手套、磨损、日期记录的终端实际报价库；
5. **磨损分层未完全自动化**：报告已显示最低 CSFloat 在售磨损，但尚未按磨损档分别计算市场价和租金；
6. **时间序列积累不足**：建议每天保存价格、求购、租金、挂单数和成交量快照，至少积累 30 天；
7. **刀型偏好先验需复核**：每季度用家族成交中位数和深度重新校准，避免主观层级固化。

建议下一阶段优先补充：

- 自有出租订单的等待时间、实际成交日租、续租率和空置天数；
- 终端实际报价截图/数据的日期、款式、磨损和接受价格；
- 国内求购队列深度及靠近求购价的真实成交；
- 四代手套按磨损档的价格、租金和成交对比。
